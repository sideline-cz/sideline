Sideline is Discord-first. Once the bot is installed, most day-to-day interactions happen in Discord — without ever opening the web app.

## What the bot does

- **Posts events** in the linked events channel with RSVP buttons.
- **Sends reminders** to players who haven't responded.
- **Syncs roles** — group membership on Sideline is mirrored to Discord roles so channel access matches reality.
- **Cleans up** old event posts according to your chosen cleanup mode.
- **Welcomes new members** — when someone joins via an invite, the bot posts a welcome embed and logs the join to a private captain channel.

## Installing the bot

See [Quick start for Admins](/quick-start/admins/#steps) for the full install flow. In short: admin → Team settings → Discord integration → **Install bot** → pick your Discord server → grant permissions.

## Channels

You link Discord channels to Sideline for specific purposes:

- **Events channel** — where event posts with RSVP buttons appear (configurable per event type: training, match, tournament, etc.).
- **Reminders channel** (optional) — where RSVP reminders and event-start announcements are posted. If unset, the bot falls back to the event's owner-group channel.
- **Late-RSVP channel** (optional) — where the bot posts a notification when a member RSVPs after the reminder has already been sent.
- **Welcome channel** (optional) — where the bot posts a public welcome embed when a new member joins via an invite.
- **System log channel** (optional) — a private captain-only channel where the bot logs every join (member, invite code, inviter, group). Keep it visible only to captains and admins.
- **Achievement channel** (optional) — where the bot posts a congratulatory embed when a member earns an achievement. Select **None** to disable achievement notifications entirely. Teams that had a welcome channel configured before this setting was introduced default to posting to that welcome channel; you can change or clear the setting at any time.
- **Weekly challenge channel** (optional) — where the bot posts the weekly challenge announcement every Monday at 09:00 in your team's timezone. The embed shows the challenge title, kind (**Házecí** for throwing, **Sportovní** for sport), and the week date range. When no channel is set the bot skips the team — enabling this feature is opt-in.

Change channel links at any time from **Team settings → Discord integration**.

### Managed channels

In addition to the purpose-specific channels above, admins and captains can create fully **managed Discord channels** from **Team → Channels**. Sideline creates and owns these channels — you assign access per group, and the bot applies the corresponding Discord permission overwrites within seconds.

See the [Channel management guide](/guides/channel-management/) for full details.

<Aside type="note" title="10-event channel limit">
  Each event channel shows at most **10 events** at a time — the most-recent past events plus the soonest upcoming events. When a new event is created or an existing one starts, the bot removes the oldest messages beyond that limit from the channel. Removed events are not deleted from Sideline; they remain visible in the web app.
</Aside>

## Slash commands

The bot responds to the following slash commands inside your linked Discord server:

- `/doprava` — post a live-updating **carpool board** in the current channel so members can offer lifts and reserve seats (Czech name; English alias: `/carpool`). Requires the `carpool:manage` permission (Admin and Captain by default). See the [Carpool board guide](/guides/carpool/) for details.
- `/event create` — create a new event (opens a modal with title, start/end time, location, and description). The modal does not include a **Location URL** field — add the link via the web app after creating the event.
- `/event list` — show your upcoming events with RSVP buttons, visible only to you.
- `/event overview` — post a persistent "Show My Events" button in the current channel (requires Manage Server permission).
- `/finance status` — check your outstanding fees. The bot replies with an ephemeral embed showing each unpaid or overdue fee with the amount owed and due date. See the [Finances guide](/guides/finances/) for details.
- `/info` — show the current bot and server version. Useful when reporting a bug or checking that the bot is up to date.
- `/summarize [messages:<1–200>] [since:<window>]` — get an AI-generated summary of the recent messages in the current channel or thread (Czech: `/shrnout`). The reply is ephemeral (visible only to you). When neither option is given the bot summarizes the last **50 messages**. Use `messages` to change the count, or `since` to summarize everything since a point in time — for example `since:24h`, `since:7d`, `since:3d12h`, or `since:2026-06-20`. The bot filters out bot messages, labels the summary with participant count and time range, and notes when the window was capped. If the AI is temporarily unavailable it replies with a brief notice instead of a partial result.
- `/summon [user:<user>] [role:<role>]` — pull a Discord user and/or all members of a Discord role into the thread where you run the command (Czech: `/přivolat`). At least one option is required. Requires **Manage Threads** permission; must be used inside a thread; the bot replies only to you.
- `/makanicko log` — log a personal physical activity. Required: activity type. Optional: duration (minutes), note, and `date` (YYYY-MM-DD, defaults to today; ±2 years limit).
- `/makanicko stats` — view your personal activity stats and streaks.
- `/makanicko leaderboard` — view the team activity leaderboard.
- `/training result` — get a private deep-link to the training result editor for a recent training event. Use the autocomplete to pick an event from the past 2 days. Only visible to members with the **Manage Events** Discord permission (captains and admins by default). Czech sub-command name: `/training výsledek`.
- `/training generate` — get a private deep-link to the **team generator** for a recent training event. The generator assigns RSVP-yes attendees to balanced teams based on Elo ratings, team size, and gender mix. Use the autocomplete to pick an event from the past 2 days, then follow the link to the web app to generate and post teams to Discord. Only visible to members with the **Manage Events** Discord permission. Czech sub-command name: `/training generovat`.

<Aside type="note" title="Where commands show up">
  Slash commands only appear in channels where the bot has permission to reply. If `/events` doesn't autocomplete, check the bot's channel permissions.
</Aside>

## Welcome message

When a new member joins your Discord server via a Sideline invite, the bot can post a welcome embed to the **welcome channel** you configure in Team settings.

The embed content comes from your **welcome message template**. Write any text and use these placeholders — the bot replaces them with real values at join time:

| Placeholder | Replaced with |
|---|---|
| `{memberMention}` | Discord @-mention of the new member (`<@id>`) |
| `{memberName}` | Display name or Discord username of the new member |
| `{inviterMention}` | Discord @-mention of whoever created the invite (`<@id>`) |
| `{inviterName}` | Discord username of the invite creator |
| `{groupName}` | Name of the group the invite targets (empty string if not group-scoped) |
| `{teamName}` | Name of your team |

The embed colour matches the group's colour (or the Sideline default blurple if the invite is not group-scoped). The group name appears as a field in the embed.

<Aside type="note" title="Welcome only fires for invite-tracked joins">
  The bot can only post a welcome and fill in inviter/group details when it successfully identifies which Sideline invite the new member used. If the invite could not be matched (e.g. the member joined through a non-Sideline invite), the welcome message is not sent.
</Aside>

### System log

Set a **system log channel** to receive a private embed for every join. The log shows the member mention, invite code used, inviter (if known), and group (if the invite was group-scoped). Keep this channel visible only to captains and admins.

## Training claim board

A configurable number of days before each training (default: 3 days), the bot posts a **claim-board message** into a dedicated **"Training claims" thread** in the owner group's Discord channel. All trainings for the same owner group share one persistent thread — the bot creates it the first time and reuses it from then on. Each individual embed shows the training's details and an orange **Claim** button.

A coach (any member of the training's owner group or a sub-group of it) can click **Claim** to mark themselves as responsible for running the training. The embed turns green and shows the coach's name with a clickable Discord mention (for example, `**Alex** (<@123456789>)`). If the training is already claimed by someone else, the button responds with a quiet "already claimed" notice visible only to you.

The current claimer can click **Release** to give up the claim, returning the embed to its unclaimed (orange) state for others to pick up.

If no one has claimed a training by the time the regular RSVP reminder fires, the bot posts an additional **unclaimed training reminder** to the owner group's channel, including a jump link back to the claim embed.

When the training starts, the claim embed is automatically removed from the thread, and the bot posts a **"Starting now"** announcement to the reminders channel. For trainings:

- If a coach claimed the training, the announcement **@-mentions the coach directly** (e.g. `<@CoachDiscordId>`).
- If no coach claimed the training (or the claimer has no linked Discord account), the announcement **@-mentions the owners-group role** and includes the message "No coach claimed this training."

On the day of the training, if the training has been claimed, the bot also posts a **"today's coach is X"** announcement to the member training channel so players know who is running the session.

<Aside type="note" title="Who can claim?">
  Only members (including coaches in sub-groups) of the training's **owner group** can claim or release a training. The Claim button silently rejects everyone else.
</Aside>

<Aside type="note" title="Configuring the lead time">
  The number of days before a training the claim-board message is posted defaults to **3**. Captains can change this in **Team settings → Coach assignment → Days before training** (range 0–30). Setting it to **0** posts the message on the day of the training. If the value is higher than the team's event horizon, the message may never appear because the training is not yet generated when the cron runs.
</Aside>

## Event roster attendance

Captains can link a named roster to a specific event (for example, a tournament squad list) from the event's detail page in the web app. Once linked, every **"yes" RSVP** from a team member triggers one of two flows depending on the **auto-approve** setting:

### Auto-approve ON

Members who RSVP "yes" are added to the roster immediately — no action needed from captains. If they later withdraw their RSVP they are removed from the roster automatically, unless they were already a roster member before the link was created.

### Auto-approve OFF (approval flow)

When a non-member RSVPs "yes", the bot posts an **Approve / Decline** embed in a dedicated thread inside the event's owner-group channel. The thread is named after the event and is created automatically the first time a request comes in for that event.

Owners can approve or decline each candidate:

- **From Discord** — click the green **Approve** or red **Decline** button on the embed. Once decided, the buttons are disabled and the embed colour updates to green (approved) or red (declined).
- **From the web app** — open the roster's detail page and use the **Pending requests** section to approve or decline each request.

<Aside type="note" title="Who can approve?">
  Only members of the event's **owner group** can click Approve or Decline. Anyone else receives a quiet ephemeral "only owners can decide" message.
</Aside>

Withdrawing a "yes" RSVP cancels the pending request and the bot removes the approval embed from the thread. When the event–roster link is removed entirely, all pending requests are cancelled and the thread is deleted.

## Group channels and roles

Each group in Sideline gets a **Discord role** automatically — this happens when the group is first created or the first time a member is added to it. The role is what controls who can see the group's channel.

A **Discord channel** is separate and only created when you explicitly request it. You have two options on the group's detail page:

- **Create channel** — the bot creates a hidden text channel and links the group's existing role to it, so only group members can see it.
- **Link existing channel** — pick a channel already in your Discord server; the bot creates and attaches the role to it.

When you unlink a channel (via the **Disconnect** button), the channel is removed from Discord (or archived/left alone, depending on your cleanup mode setting), but the group's Discord **role is kept**. This means members retain their role assignment and it can be linked to a new channel later without losing the membership data.

Deleting a group removes both the channel and the role.

## Roster Discord role sync

Each roster that has a Discord channel also has an associated Discord role. When members are added to the roster, they are given the role automatically. If the role gets out of sync — for example after a roster is reactivated — you can re-apply it manually.

### Per-roster sync

Open the roster detail page and click **Sync with Discord**. The bot will:

1. Ensure the roster's Discord role exists (creating it if necessary, or reusing the existing one).
2. Add every current roster member to that role.

<Aside type="note" title="Add-only in this release">
  The sync button currently **adds** members to the role and **does not remove** anyone. If a former member still holds the role after being removed from the roster, remove them from the Discord role manually. Automatic pruning of stale role-holders is planned for a future release.
</Aside>

The button is available to any member with the **roster:manage** permission (Captain and Admin by default). The sync runs asynchronously — the page returns a count of members queued immediately, and the bot applies the role changes within seconds.

### Team-wide re-sync

If multiple rosters have drifted out of sync at once — for example after a Discord outage — use the **Re-sync roster role members** button on the rosters list page. It sweeps all active rosters that already have a Discord role and re-queues the member sync for each one.

- Processes up to 50 rosters per click. If your team has more than 50 eligible rosters, click again to process the next batch. The button shows how many rosters remain.
- This tool re-adds missing members; it does not create roles for rosters that have none, and it does not remove former members from existing roles.
- Requires the **roster:manage** permission (Captain or Admin).

## Roster channel category

When Sideline creates a Discord channel for a roster (on creation or reactivation), it can place that channel inside a Discord category you choose.

Set the **Roster category** dropdown in **Team settings → Discord integration → Roster category**. The setting accepts any category channel in your Discord server, or **None** (the default) to place roster channels at the guild root without a category.

When a roster channel is created and a category is configured, the bot passes `parent_id` to Discord. If the configured category has been deleted or is otherwise unreachable, the bot falls back to guild-root placement automatically — no manual intervention is needed.

<Aside type="note" title="Existing channels are not moved">
  Changing this setting only affects new roster channels created after the change. Existing roster channels are not moved. If you want to move an existing channel, do it manually in Discord or use the **Team → Channels** archive/restore flow.
</Aside>

## Cleanup modes

Old event posts accumulate. You pick a cleanup strategy per team:

- **Do nothing** — posts stay forever.
- **Delete** — posts are removed a configurable time after the event ends.
- **Archive** — posts are moved to an archive channel you configure.

## Troubleshooting

<Aside type="caution" title="Bot doesn't respond">
  Most common cause: the bot doesn't have permission to send messages or read the channel. Check Discord server settings → **Integrations → Sideline → channel permissions**.
</Aside>
