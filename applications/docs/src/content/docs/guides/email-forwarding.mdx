## How it works

When an email arrives at the configured inbound address:

1. Sideline stores the email and asks the AI to produce two summaries: a **short summary** (one sentence + key bullet points, shown in Discord) and a **detailed summary** (fuller breakdown for coach review).
2. The bot posts an **approval request** to the private coach channel: an amber embed with the short summary and an email metadata (From/Subject/Received), followed by a blurple embed with the detailed summary.
3. A coach reviews both summaries. They can open the **Email detail page** (`/teams/{teamId}/emails/{emailId}`) in the web app to edit either summary, then click **Approve** or **Reject** in Discord.
4. On approval, the **short summary** is posted to the team channel. Team members can click **Detailed summary** or **Original email** to see the full text as an ephemeral (visible only to them) paginated preview.
5. On rejection, the original email text is posted to the team channel instead.
6. Attached files are stored and downloadable from the Email detail page by any team member.

## Who can do what

| Action | Permission required |
|---|---|
| Configure email forwarding settings | `team:manage` |
| Approve or reject an email | `team:manage` |
| Edit the short or detailed AI summary | `team:manage` |
| View an email and download attachments | Any team member |

## Ingestion methods

Sideline supports two ways to receive emails. You can use either or both at the same time.

<Tabs>
  <TabItem label="Inbound webhook">

Your email provider forwards a copy of each incoming message to a unique webhook URL generated by Sideline. Most transactional email services (e.g. Postmark, SendGrid Inbound Parse, Mailgun) support this forwarding mode out of the box.

<Steps>

1. Go to **Team settings → Email forwarding** and enable the feature.
2. Set the **coach channel** (where approval requests appear) and the **target channel** (where approved or rejected emails are posted).
3. Optionally add **allowed senders** to filter which sender addresses are accepted (matched against each email's *From*); leave the list empty to accept all.
4. Copy the **inbound webhook URL** (generated after saving) and configure your email provider to forward to it.
5. Rotate the token with **Regenerate token** if the URL is ever compromised.

</Steps>

  </TabItem>
  <TabItem label="IMAP polling">

Connect Sideline directly to a mailbox using IMAP. Sideline polls the mailbox every 5 minutes and ingests any new messages. This is useful when you control the email account and do not want to set up a forwarding service.

<Steps>

1. Go to **Team settings → Email forwarding** and enable the feature (if not already done).
2. Set the **coach channel** and **target channel** (shared with the webhook method).
3. In the **IMAP** section, enable IMAP polling and fill in:
   - **Host** — your mail server hostname (e.g. `imap.gmail.com`)
   - **Port** — typically `993` for IMAPS or `143` for STARTTLS
   - **TLS** — enable for direct TLS connections (port 993); disable only for STARTTLS
   - **Username** — the full email address or account login
   - **App password** — the account password or, for services like Gmail, a dedicated app password. This is stored encrypted and never shown again after saving.
   - **Folder** — the mailbox folder to poll (leave blank to use `INBOX`)
4. Save the settings.

</Steps>

<Aside type="note" title="Cold start behaviour">
  Only mail received **after** you save the IMAP settings is processed. Sideline records the current highest message UID on the first successful poll and skips everything before it. No historical emails are ingested.
</Aside>

<Aside type="tip" title="Use an app password">
  For Gmail and other providers that support two-factor authentication, create a dedicated app password instead of using your main account password. App passwords can be revoked independently without affecting your main account.
</Aside>

  </TabItem>
