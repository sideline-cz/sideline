By the end of this page you'll have the Sideline bot installed on your Discord server, linked to a Sideline team, with channels and permissions configured.

## Before you start

- You need **Manage Server** permissions on the Discord server.
- You need to be signed in to the Sideline web app with the Discord account that will own the team.

## Steps

<Steps>

1. **Create a team.**

   From the dashboard, click **New team**. Pick a name and a short description. You become the team's first admin automatically.

2. **Invite the Sideline bot.**

   From **Team settings → Discord integration**, click **Install bot**. Discord's OAuth flow asks which server to add the bot to — pick yours. Grant the permissions it requests (send messages, manage roles, read member list).

3. **Link channels.**

   Sideline needs to know which Discord channels it should post into. At minimum, pick an **events channel** — that's where RSVP prompts appear. Optionally pick a **reminders channel** if you want late-RSVP pings in a separate place.

4. **Configure groups and roles.**

   Decide how your team is subdivided (see [Key concepts](/introduction/key-concepts/#group)). Create groups in the web app; Sideline can auto-create matching Discord roles so channel access stays in sync.

5. **Invite captains.**

   Use **Roster → Invite** to generate invite links scoped to the **captain** role. Send them to your captains over Discord DM.

</Steps>

<Aside type="caution" title="Permission model">
  Admins can do anything. Captains can manage events and rosters. Players can only manage their own RSVPs. Changing someone's role is immediate and retroactive — archived events keep their old RSVP history.

  There's also a built-in `Treasurer` role for delegating finance authority (creating fees, recording payments) to a non-admin member.
</Aside>

## After setup

Once the bot is installed and linked:

- Captains create events; Sideline posts them automatically.
- Players join via your invite links; Sideline matches them to Discord accounts.
- If someone leaves (or is kicked/banned from) your Discord server, Sideline automatically deactivates their membership and removes them from all groups and rosters. Their event and attendance history is preserved, and an admin can reactivate them later. The last active admin of the team is always protected from automatic deactivation.

## Onboarding new teams (Global Admin)

If you are a **global Sideline admin**, you can mint one-time **onboarding links** that let a designated captain complete their team's setup without needing a Sideline account first. You also have **read-only access to every team** in the installation — you can browse any team's members, rosters, roles, finance overview, and activity stats without being enrolled as a member. You cannot edit or delete data in teams you are not a member of.

A user is a global admin when **either** of the following is true:

- Their account has the global-admin flag set in the database. The **first person to sign in** to a fresh Sideline installation receives this flag automatically.
- Their Discord ID is listed in the `APP_GLOBAL_ADMIN_DISCORD_IDS` server environment variable (useful for adding admins without a code change or for pre-existing deployments).

<Aside type="tip" title="First-time setup">
  On a brand-new installation, just sign in with Discord. You will be recognised as the global admin immediately and redirected to **Administration → Team onboarding** where you can start issuing onboarding links.
</Aside>

### Manage global admins

Navigate to **Administration → Global admins** (`/admin/global-admins`) to view the current list of global admins and promote or demote users.

**Grant admin access**

Enter the Discord user ID of the person you want to promote into the **Grant admin** form and click **Grant**. The user must have signed in to Sideline at least once — the Discord ID must match an existing account. See [Find a captain's Discord user ID](#find-a-captains-discord-user-id) for how to copy a Discord ID.

**Revoke admin access**

Click **Revoke** next to a user in the list and confirm the dialog. A few safeguards apply:

- You cannot revoke your own admin status.
- You cannot revoke a user whose Discord ID is managed via the `APP_GLOBAL_ADMIN_DISCORD_IDS` environment variable (they show an **Env-managed** badge). Ask your server operator to remove them from the env var instead.
- You cannot revoke the last remaining global admin — at least one must always exist.

<Aside type="caution" title="Env-managed admins">
  Users listed in the `APP_GLOBAL_ADMIN_DISCORD_IDS` server environment variable cannot be revoked from the web UI. Their access is controlled by the server operator. The web page shows them with an **Env-managed** badge and no Revoke button.
</Aside>

### Mint an onboarding link

<Steps>

1. Navigate to **Administration → Team onboarding** in the web app.

2. Click **Create onboarding link** and fill in the form:

   | Field | Description |
   |---|---|
   | **Proposed team name** | Suggested name for the team. The captain can change it during setup. |
   | **Captain's Discord user ID** | Only this Discord account can use the link. See below for how to find it. |
   | **Link expires after** | Choose 24 hours, 3 days, or 7 days. The link cannot be used after it expires. |

3. Click **Create link**. The link is shown **once only** — copy it immediately and send it to the captain over Discord.

</Steps>

<Aside type="caution" title="One-time display">
  The generated URL is shown only at creation time. Sideline does not store the full URL. If you lose it, revoke the token and create a new one.
</Aside>

### Find a captain's Discord user ID

1. Open Discord and go to **User Settings → Advanced**.
2. Enable **Developer Mode**.
3. Right-click (or long-press on mobile) the captain's profile anywhere in Discord.
4. Select **Copy User ID**.

The ID is a long numeric string, for example `123456789012345678`.

### Manage existing tokens

The **Existing tokens** list shows all tokens with their status:

| Status | Meaning |
|---|---|
| **Active** | The link can still be used. |
| **Used** | The captain completed onboarding; the link is consumed. |
| **Expired** | The expiry window passed before the captain used it. |
| **Revoked** | A global admin manually revoked the link. |

To cancel an unused link, click **Revoke** next to it and confirm.
